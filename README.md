# Y.AddURL

**Y.AddURL** — это компактный, надежный и безопасный инструмент для автоматического управления переобходом URL через **Yandex Webmaster API v4**. Приложение позволяет массово добавлять сайты и ссылки, распределять квоты на переобход и следить за процессом в режиме реального времени.

## ✨ Возможности

* **Массовое управление:** Добавление сайтов и пулов ссылок списком или через загрузку файлов.
* **Yandex Webmaster API:** Автоматическое добавление хостов, проверка прав и отправка страниц на переобход.
* **Справедливые квоты (Fair Quota):** Интеллектуальный алгоритм "water-filling" распределяет доступную суточную квоту (Recrawl Quota) между сайтами пропорционально количеству их необработанных ссылок.
* **Поддержка множества ключей:** Управление множеством API-ключей (OAuth Yandex) с привязкой конкретных сайтов.
* **Асинхронные логи:** Удобная панель управления с real-time логированием работы крона.
* **Безопасность включена:** Защита от SQL-инъекций, CSRF токенизация, строгая защита от XSS, транзакционная целостность БД.

## 🚀 Установка и настройка

Проект использует SQLite, поэтому не требует установки отдельного SQL-сервера.

### 1. Требования
* **PHP 8.0** или выше
* Расширения PHP: `sqlite3`, `pdo_sqlite`, `curl`, `mbstring`
* Веб-сервер (Apache с поддержкой `mod_rewrite` или Nginx)

### 2. Развертывание
1. Склонируйте репозиторий в папку на вашем веб-сервере.
2. Убедитесь, что веб-сервер направляет все запросы на `index.php` (для Apache уже настроен `.htaccess`), для nginx конфиг такой же как и для wordpress.
3. Обязательно выдайте веб-серверу полные права **на чтение и запись** (write permissions) для папки `/DATA` — в ней хранятся файлы баз данных (`db.sqlite`, `log_db.sqlite`) и другой кэш. На файлы `db.sqlite`, `log_db.sqlite` и `psl_cache.php` тоже надо выдать права на запись.  Без этого приложение не сможет работать.
4. Откройте файл `config.php` и измените стандартный пароль (`_PASSWD_`) для доступа к админс-панели на свой.

### 3. Настройка крон-задач (Cron)
Вся обработка очередей, взаимодействие с API Яндекса и распределение квот происходит в фоновом режиме через Cron. Добавьте следующие задачи в планировщик вашего сервера:

```bash
# Основная обработка очереди (каждую минуту)
* * * * * /usr/bin/php /абсолютный/путь/до/сайта/index.php cron=queue >> /путь/до/cron.log 2>&1

# Обновление кэша Public Suffix List (например, 1 раз в месяц)
0 0 1 * * /usr/bin/php /абсолютный/путь/до/сайта/index.php cron=update_psl >> /путь/до/psl.log 2>&1
```
*(Замените `/usr/bin/php` и пути на актуальные для вашего сервера).*

## 📊 Статусы сайтов

В панели управления сайты могут иметь один из следующих статусов работы:

* `1` — **Все ОК, готов к работе.** Сайт верифицирован, и скрипт активно отправляет ссылки на переобход.
* `0` — **Остановлен.** Отправка ссылок приостановлена пользователем (пауза).
* `3` — **Сайт не верифицирован.** Сайт добавлен в панель, но права на него в Yandex Webmaster еще не подтверждены. Крон будет периодически пытаться проверить права.
